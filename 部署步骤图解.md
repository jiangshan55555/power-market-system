# 🎨 GitHub Desktop + Vercel 部署图解

## 📋 完整流程一览

```
┌─────────────────────────────────────────────────────────────┐
│                     部署流程总览                              │
└─────────────────────────────────────────────────────────────┘

第一步：安装 GitHub Desktop
    ↓
    📥 下载：https://desktop.github.com
    📥 安装并登录 GitHub 账号
    
第二步：添加项目到 GitHub Desktop
    ↓
    📁 File → Add local repository
    📁 选择：power-market-system 文件夹
    
第三步：提交代码
    ↓
    ✍️  填写提交信息
    ✅ 点击 "Commit to main"
    
第四步：发布到 GitHub
    ↓
    🚀 点击 "Publish repository"
    ⚙️  配置仓库名称和可见性
    ✅ 确认发布
    
第五步：连接 Vercel
    ↓
    🌐 访问 https://vercel.com
    🔑 使用 GitHub 账号登录
    
第六步：导入项目
    ↓
    ➕ Add New → Project
    📦 选择 power-market-system
    ⚙️  配置项目设置
    
第七步：部署
    ↓
    🚀 点击 "Deploy"
    ⏳ 等待 1-3 分钟
    
第八步：完成！
    ↓
    🎉 获得在线地址
    🌐 访问你的网站
```

## 🎯 关键提示

### ✅ 推荐做法

1. **提交信息规范**：
   - `feat:` 新功能
   - `fix:` 修复bug
   - `style:` 样式调整
   - `docs:` 文档更新

2. **定期推送**：
   - 每完成一个功能就提交
   - 避免一次性提交太多修改

3. **查看部署日志**：
   - 如果部署失败，查看 Build Logs
   - 根据错误信息调整代码

### ❌ 避免的错误

1. ❌ 不要提交 `node_modules` 文件夹
2. ❌ 不要提交 `.env` 等敏感文件
3. ❌ 不要在生产环境使用 `npm run dev`
4. ❌ 不要忘记推送到 GitHub（只提交不推送）
